## 1. 真實無人機足球規則摘要(查證來源見文末)

無人機足球(Drone Soccer)是 FAI(國際航空運動聯合會)認可的對抗性團隊運動,源自韓國。重點規則:

| 項目 | 規則 |
|---|---|
| **球 = 無人機本身** | 每台無人機包在球形保護框裡,飛機就是「球」。 |
| **隊伍人數** | 每隊 3–5 人(我們要做 **3v3**)。 |
| **前鋒(Striker)** | 每隊指定**一台**為前鋒,**只有前鋒**飛過對方球門得分。 |
| **其他隊員** | 當前鋒的開路/掩護,或防守自家球門。 |
| **得分** | 前鋒的機體**完整、朝正向**穿過對方圓形球門 = 1 分。非前鋒穿過不算分(也不罰)。 |
| **得分後** | 該隊**全員須先退回自己半場**(過中線)才能再次得分。 |
| **賽制** | 一場分 **3 局,每局 3 分鐘**;贏 2 局者勝。平手進驟死賽,先進球者勝。 |
| **場地** | 矩形籠子,約 **16m × 7m**(長短邊約 2:1),四周有護網。 |
| **接觸** | 是**接觸性運動** —— 可以用機體去**撞、擋、卡位**對方(這是策略核心)。 |
| **倒地機** | 對手可在墜機上方盤旋,但不可碰觸該墜機。 |
| **球門** | 兩端各一個**圓環球門**(架高)。前鋒穿過得分。 |
| **級別** | 依機體大小分級;**20cm 級**(球框直徑約 20cm)是常見的青少年/教育級別 ← **我們要做這級**。 |

## 2. 我們的版本:3v3、20cm 級(在模擬器裡)

核心對應:**「無人機就是球、前鋒穿門得分」這套機制,跟我們現有的「玩家無人機 + 鑽圈」幾乎天生吻合** —— 大量可重用大亂鬥的程式。

### 2.1 場地與球門（已定:3×3×6 小場地）
- **場地尺寸**:**6m(長,兩球門連線方向)× 3m(寬)× 3m(高)** 的封閉空間(20cm 級的緊湊場地)。
  - 六面都是邊界(地面 + 天花板 3m + 四面牆),飛機撞牆會被擋住(沿用 solid 方塊的 AABB 牆面)。
  - 中線在長軸 3m 處,將場地分為各 3m 深的兩半場。
  > 軸向解讀:**長 6m = 兩球門連線方向、寬 3m、高 3m**。若你原意的長寬高對應不同,跟我說我改。
- **兩個圓形球門**:沿**長軸**各架在 6m 的兩端牆面;**門中心高約 1.5m**(配合 3m 天花板,也與站位視角等高)。
  - **球門位置關係**:從任一隊的窄邊站位往場內看,**近端門(己方半場那端牆)**= 要防守的門;**遠端門(對方半場那端牆)** = 要攻向的門。**兩個門環沿長軸排列,站在窄邊即可同時看到兩個門**,沒有「球門在背後」的情形。
  - 這與真實賽制相同:選手面朝場內操作,視線穿越整個 6m 長軸,近端守門、遠端攻門。
- **球門尺寸(配合小場地縮小)**:內徑約 **1.2m**(讓 20cm 級飛機好穿、又不會太好進)。
  > 註:模擬器尺度可微調,以「好穿、好玩、防守有意義」為準。
- **空間很緊湊**:6×3×3 放 6 台(3v3)會很擠、貼身纏鬥 —— 這正是無人機足球的混戰感,但**模擬器要把飛行速度/靈敏度調溫和**,否則在小空間很難控。
- **隊色**:藍隊 / 紅隊(各 3 台);前鋒再加明顯標記(皇冠光環或更亮)。

> **待確認 — 球門相關設計點**（正確描述確定後，實作前需釐清）：
> 1. **得分方向偵測用哪個門**：計分只看「前鋒穿越遠端門(對方的門)」，近端門(己方的門)的穿越事件要不要偵測並回傳給伺服器(例如顯示「防守被破」提示)?或完全忽略?
> 2. **門中心高 1.5m 與視角等高**:攝影機高度同樣設 1.5m,站位視角正對門環中心;確認這是預期效果(視線穿過兩個門的圓心)。
> 3. **半場/中線與兩門的關係**:中線位在 6m 長軸的正中(距兩端門各 3m),「得分後全隊退回半場」= 全隊越過中線回到自己那 3m 半場;確認這條中線不是「退到己方近端門後面」。

### 2.2 角色(3v3)
- 每隊 3 台:**1 前鋒 + 2 防守/助攻**。
- 前鋒:**固定 / 隊伍自選**(不隨機),每隊恰 1 名;前鋒機體有特殊外觀(顏色更亮 / 頭頂標記)。
- 只有**前鋒穿過對方球門**才算分。

### 2.3 計分與規則(伺服器權威)
- 前鋒**朝正向完整穿過對方球門(即自己視角的遠端門)** → +1 分(沿用鑽圈的穿越偵測 + 方向判定 `faceYaw` 概念)。
- 得分後:**該隊全員需退回自家半場(過中線)** 才能再得分 → 伺服器檢查、未退回前的穿門不計。
- **賽制(已定)**:**1 局 3 分鐘,進球多者勝**(課堂節奏快)。正式 3 局制列為後續選項。
- **開賽**:沿用大亂鬥的「老師按開始 → 全場 3-2-1 倒數起跑」。

### 2.4 接觸/碰撞(關鍵差異,需決定)
真實無人機足球**可以互撞、卡位**,這是策略精髓。但我們目前的大亂鬥是**玩家互不碰撞(會穿過)**。所以足球模式要不要做「機體互撞」是**最大的設計分歧點**:
- **A. 無碰撞(最省、最快)**:純比「誰先把前鋒送穿門」,防守靠擋在門前的視覺/位置壓迫,但不能真的撞開。→ 像「3v3 搶穿門競速」。
- **B. 有碰撞推擠(像真足球)**:加玩家↔玩家碰撞(推開/彈開),防守可以卡位、撞偏前鋒。→ 最有真實感,但要做多人碰撞物理(伺服器權威或客戶端 + 校正),工程較大。
- **建議**:先做 **A 版**能玩,之後再加 **B 版**碰撞(就像你之前說「相撞版有空再試」)。

### 2.5 視角 / 攝影機(已定:窄邊定點視角,模擬真實場邊操作)

真實無人機足球:**選手站在自己這端的窄邊,面朝場內操作**。我們照這個來:

- **站位**:每隊 3 人站在場地的**窄邊(3m 寬那一邊)**,一隊一邊;**紅隊一端、藍隊另一端**,相距 6m(長軸)。
  - **球門在哪裡**:兩個門環沿長軸排列。從你的窄邊站位往場內看,**近端(你這側的端牆)那個門 = 你要防守的門**;**遠端(對方那側的端牆)那個門 = 你要攻向的門**。**兩個門都在你的正前方視野裡**,不存在「球門在背後」的情形。
- **視角**:**由窄邊看入** —— 攝影機固定在自己隊伍的窄邊底線稍外側、**高度 1.5m**(與球門中心等高),水平**朝場中央/對方那端**看。視線沿長軸延伸:先遇到近端門環(己方要守的門),穿過後看到對方陣地,最後是遠端門環(要進球的門)。
  - 紅隊與藍隊**各自的視角相反**(紅看向藍端、藍看向紅端),兩隊看到的是「面對面」的畫面。
  - 這是**定點場邊視角**(像場邊操作手),**不跟著自己的飛機跑**;飛機飛出去會在畫面中變遠變小(6m 場地不算遠,可接受)。
- **實作對應**:
  - 攝影機位置依「我屬於哪一隊」決定:紅隊 `z≈+4`(底線外一點)看向 −z;藍隊 `z≈−4` 看向 +z;`y=1.5`。(座標以長軸為 z、場地中心為原點;實作時對齊。)
  - 預設用這個**窄邊隊伍視角**;仍可用 `C` 鍵切到 **FPV(第一視角)** 當輔助看法。
  - 與一般關卡/大亂鬥的「跟隨第三人稱」不同 —— 足球是**固定隊伍視角**,要在進入足球模式時切換攝影機邏輯。

### 2.6 單人練習模式

> **目的**:讓學生一個人也能上場熟悉足球場地、球門位置與穿門手感,不需等人湊隊。練習項目由淺入深,對應 3v3 正式賽會用到的核心技巧(方向感、穿門精準度、半場節奏)。

#### 2.6.1 場地與視角(與多人賽完全相同)

- 沿用同一個 6×3×3m 封閉場地、兩端球門(門中心高 1.5m、內徑約 1.2m)、中線。
- 攝影機維持**窄邊定點視角 1.5m**,學生站在窄邊往場內看,近端門=近、遠端門=遠。
- 單人時不分隊、無對手,僅有自己的一台無人機在場上。

#### 2.6.2 練習項目序列(由淺入深)

| 關卡 | 名稱 | 目標 | 過關條件(建議) |
|---|---|---|---|
| P-1 | **熟悉場地** | 從出生點(近端門前方)飛到遠端門前方,再飛回來,感受 6m 長軸距離 | 30 秒內完成來回一趟(不需穿門) |
| P-2 | **單次穿門** | 從中場起飛,朝正向完整穿越遠端門 1 次 | 成功穿越遠端門 1 次(方向判定同正式賽) |
| P-3 | **連續穿門 × 3** | 連續穿越遠端門 3 次(每次穿越後飛回中場重整) | 60 秒內完成 3 次 |
| P-4 | **計時衝刺** | 計時:從近端門前方起飛 → 穿越遠端門 → 飛回近端門前方停定 | 紀錄最佳時間(無過關門檻,鼓勵自我挑戰) |
| P-5 | **靜態障礙穿門** | 遠端門前方放置 1~2 個靜態假人(模擬防守機),繞過後穿門 | 45 秒內穿門成功 1 次 |
| P-6 | **兩端來回** | 模擬「得分後退半場」節奏:穿遠端門 → 退回近端門前方(過中線)→ 再穿遠端門,重複 3 回合 | 完成 3 回合(每回合退回後自動解鎖下一次穿門) |
| P-7 | **限時多穿門** | 3 分鐘(一局時長)內盡量多穿遠端門 | 無強制過關,顯示得分數與最佳紀錄 |

> 難度說明:P-1～P-2 為操控入門;P-3～P-4 為穩定度訓練;P-5 為策略感知入門;P-6 直接對應正式賽「得分後退半場」規則;P-7 模擬完整一局節奏。

#### 2.6.3 計分與記錄

- **計時**:各關卡顯示倒數或正計時 HUD(沿用現有計時 HUD)。
- **最佳紀錄**:P-4 與 P-7 儲存個人最佳(本機 localStorage 即可,不需伺服器)。
- **穿門計數**:畫面右上方顯示「本次 X 次 / 最佳 Y 次」(P-3、P-7 適用)。
- **過關提示**:達到過關條件時顯示過關動畫(沿用大亂鬥/關卡的慶祝提示);未達成則顯示「再試一次」與本次成績。
- **無伺服器同步**:單人練習不接入多人伺服器,避免干擾正在進行的課堂賽局。

#### 2.6.4 與多人賽的關係

| 面向 | 多人賽(3v3) | 單人練習 |
|---|---|---|
| 場地 / 球門 / 視角 | 6×3×3m,窄邊 1.5m,兩球門 | **完全相同(共用場景資源)** |
| 伺服器連線 | 需要(伺服器權威) | **不需要(純本機)** |
| 分隊 / 前鋒 | 是 | **否(無隊伍概念)** |
| 穿門方向判定 | 伺服器驗證 faceYaw | **客戶端驗證(邏輯相同,省去網路往返)** |
| 對手 | 3 名真實玩家 | 無(P-5 用靜態假人) |
| 計分 HUD | 藍 X:Y 紅 + 局數 + 時間 | 穿門計數 + 計時(簡化版) |

#### 2.6.5 可重用的現有程式模組

| 現有模組 | 如何套用到單人練習 |
|---|---|
| **鑽圈穿越 + 方向判定(faceYaw)** | 直接套用,偵測「朝正向完整穿越遠端門」,邏輯與正式賽相同 |
| **pass zone 分步過關** | P-1「先到達遠端門前方、再飛回」可拆成兩個 pass zone 依序觸發 |
| **3-2-1 倒數** | 每個關卡開始前沿用,維持學生熟悉的起跑節奏 |
| **計時 HUD** | 直接重用,顯示關卡剩餘時間或正計時 |
| **過關/慶祝提示** | 達到過關條件時直接觸發,不需改寫 |
| **靜態假人渲染** | P-5 的障礙用現有他人分身渲染邏輯,設為固定座標不動即可 |

#### 2.6.6 設計決策

**已拍板 ✅**
- **關卡解鎖**:**七關一次全開,學生自選**(不強制循序解鎖;不需關卡進度狀態管理,課堂上學生想練哪關就練哪關)。
- **P-6 退半場判定**:**用「過中線」** —— 與正式賽同邏輯(中線在長軸正中 z=0,過中線才算退回),可直接重用正式賽的 pass zone 判定,兩邊一致。

**仍待確認**
1. **進入方式**:從大廳選「單人練習」頁面,還是在等待多人賽時的「一個人先練」捷徑?(影響 UI 動線)
2. **靜態假人數量與位置(P-5)**:幾個、放哪裡?建議 1 個放遠端門前方 1m 處,可微調。
3. **紀錄儲存範圍**:只存本機,還是登入後同步伺服器(顯示班級最佳排行)?

## 4. 已拍板的設計決策 ✅

1. **賽制**:**課堂簡版 —— 1 局 3 分鐘,進球多者勝**(平手可加時或並列,實作時再定)。
2. **碰撞**:**先做 A 版(玩家互不碰撞)**;B 版撞擊/卡位列為後續階段。
3. **分隊**:**兩者都要** —— 伺服器**預設自動平均分**藍/紅,**老師後台可手動調整/指定**某人到哪隊。
4. **前鋒**:**不隨機,固定/隊伍自選** —— 每隊**恰 1 名**前鋒,由老師指定或隊伍自己選定(進場時或開賽前決定,整局固定)。前鋒機體有明顯標記。

### 仍待細談(實作時再定,不擋設計)
- 人數湊不滿 3v3 時怎麼辦(自動縮成 2v2 / 多的觀戰或排隊換人)。
- 場地已定 **6×3×3m**、球門內徑約 1.2m / 中心高 1.5m —— 細部尺寸再依手感微調。
- 得分後退半場:嚴格「全員退回」還是簡化「前鋒退回」即可。
- 前鋒「自選」的介面:老師後台指定 vs 學生端自己按「我當前鋒」(需限制每隊只能 1 個)。

---

## 5. 建議分階段實作(等規則拍板後)

- **階段 1（MVP,A 版無碰撞）**：分隊 + 前鋒 + 兩球門 + 前鋒穿門計分 + 比分 HUD + 1 局 3 分。先能 3v3 對打進球。
- **階段 2**：半場重置規則、3 局制 + 局數/換邊、前鋒標記與隊色優化、得分特效。
- **階段 3（B 版）**：玩家↔玩家碰撞推擠(卡位、撞偏),讓防守真的有用。
- **階段 4**：老師後台足球控制(開賽、分隊、比分、暫停)、回放/最佳進球。
