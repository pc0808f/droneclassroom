## 目標

實作足球**得分核心規則**：前鋒朝正向完整穿過**對方球門**才得分、得分後該隊須**退回半場（過中線）** 才能再得分、雙方比分顯示在 HUD（**藍 X : Y 紅**）。

## 背景

- **穿門 + 方向判定可重用鑽圈**：一般關卡旋轉鑽圈已有 `faceYaw`（`missionRings[i].faceYaw`）判定「朝正向穿過」；足球穿門用同一套概念。
- 計分必須**伺服器權威**（T-501 已收 `soccer_pos` / `soccer_goal`，本 task 補上計分驗證）。
- 半場規則：中線 `z=0`，得分後全隊（MVP 簡化為**前鋒**）須過中線回自家半場才能再得分（T-501 已有 `armed` 旗標位）。

## 描述

### 1. 前鋒穿門偵測（client 偵測 → 上報，server 驗證）
- client 每幀檢查**自己**是否「朝正向完整穿過對方球門（自己視角的遠端門）」：
  - 穿越偵測：沿用單人足球的穿門偵測（`soccerTick` 的 `prevZ` 跨越門平面）。
  - 方向判定：機頭 yaw 對齊門法向（沿用 `faceYaw` 容差概念），避免倒退穿門 / 擦邊。
  - 機體需從門環內徑內通過（用門半徑判定，沿用 `_soccerBVH` / 半徑檢查）。
- 命中 → 送 `soccer_goal { goal: 'far' }`（含必要座標/時間戳）。
- **server 驗證才算分**（防作弊、統一裁決）：
  - 送出者必須 `striker=true`（非前鋒穿門 → 忽略，不計不罰）。
  - 穿的必須是**對方的門**（依 `team` 判定哪端是對方門）。
  - 該隊 `armed[team]===true`（未退半場不算 — 見第 3 點）。
  - 通過 → `scores[team] += 1`、`armed[team] = false`、廣播 `soccer_scores` + `soccer_goal_ok`（給進球特效/音效）。

### 2. 比分 HUD（藍 X : Y 紅）
- 賽局中顯示比分板：**藍 X : Y 紅** + 本局剩餘時間（由 `endTime` 倒數）+ 倒數/結束狀態。
- 自己是前鋒 / 防守、屬藍 / 紅，HUD 標明。
- 進球瞬間：比分跳動 + 簡單特效/音效（沿用過關慶祝/音效模組；完整特效＝階段 2）。
- 結束（`soccer_end`）：顯示勝負（藍勝 / 紅勝 / 平手）。

### 3. 半場重置規則（得分後須過中線）
- 得分後該隊 `armed=false`。
- server 在 tick 用各玩家 `soccer_pos` 的 z 判定：當該隊**前鋒**（MVP 簡化）越過中線回到自家半場 → `armed[team]=true`，廣播 `soccer_scores`（armed 更新）。
- `armed=false` 期間，該隊前鋒穿門**不計分**。
- client HUD：`armed=false` 時對該隊顯示「⚠️ 先退回半場才能再得分」。

### 4. 邊界處理
- 同一次穿越只計一次（去抖：穿門後短暫鎖定，或用 armed 自然擋住二次）。
- 近端門（己方門）被穿越：MVP **忽略**（不計、不罰）；是否顯示「防守被破」提示列為選項（見設計文件待確認）。

## 驗收標準

1. **前鋒穿對方門 +1**：前鋒朝正向穿遠端門 → 比分該隊 +1。
2. **非前鋒不得分**：防守手穿門 → 不計分、不罰。
3. **方向/完整性**：倒退穿門、擦邊未過內徑 → 不計分。
4. **半場重置**：得分後該隊未過中線前再穿門 **不計**；前鋒過中線後恢復可得分。
5. **比分 HUD 正確**：藍 X : Y 紅 即時更新、剩餘時間倒數、結束顯示勝負。
6. **伺服器權威**：偽造 `soccer_goal`（非前鋒 / 對自己門 / 未 armed）一律不計分。
7. **無 console error / 無重複計分**。

## Sam 怎麼驗收

1. 進多人足球、開賽（測試可 1~2 人，指定自己為前鋒）。
2. 前鋒朝正向穿遠端門 → 比分 +1、有進球提示。
3. 不退回半場直接再穿 → **不加分**，HUD 提示「先退回半場」。
4. 飛回過中線後再穿 → 又 +1。
5. 用防守手穿門 → 不加分。
6. 倒退/擦邊穿門 → 不加分。

## 風險 / 注意

- **場地常數一致**：穿門判定的門 z / 半徑 / 中線必須與 T-501、T-502 同一份，否則 client 偵測與 server 驗證對不上。
- **去抖**：避免一次穿越被連續幀計成多分。
- **方向判定容差**：太嚴學生穿不過、太鬆擦邊也算 — 依手感調（沿用 `faceYaw` 容差參數）。
- **client 偵測 + server 驗證**雙層：client 只負責「我覺得我進了」上報，最終以 server 為準（與單人練習的純 client 判定不同）。
